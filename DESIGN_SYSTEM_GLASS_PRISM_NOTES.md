## Color palette

- Background:
  - very light mist blue / cool gray
- Primary text:
  - deep navy
- Secondary text:
  - muted slate blue
- Accent colors:
  - teal
  - aqua
  - mint
  - restrained blue-violet
  - amber for warning / review emphasis

## Typography scale

- Page title / hero title:
  - largest and highest contrast
- Section title:
  - medium-large with tighter spacing
- KPI values:
  - very large and prominent
- Card title:
  - medium and strong
- Labels / metadata / helper text:
  - smaller, lighter, and visually secondary

## Card hierarchy rules

- A card should usually contain:
  - one label
  - one value or action
  - one short support line only if needed
- Main numbers must dominate visually.
- Labels should never compete with values.
- Long descriptions inside cards should be avoided.

## Glass / prism styling rules

- Use translucent white surfaces with restrained blur.
- Use soft cool shadows instead of heavy dark shadows.
- Use subtle reflective borders rather than thick outlines.
- Use prism-tinted gradients only as a soft depth cue.
- Keep decoration behind the information, not in front of it.

## What should be large vs small in the UI

- Large:
  - dashboard hero title
  - KPI values
  - key counts
  - primary actions
- Medium:
  - section titles
  - card titles
  - selected state information
- Small:
  - labels
  - helper text
  - metadata chips
  - explanatory notes
