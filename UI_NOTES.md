# UI Notes

## Daily Schedule
- ใช้ `ตาราง` เป็นมุมมองเริ่มต้น เพราะเหมาะกับงานตรวจเวรรายวันบนหน้าจอคอมมากกว่าการ์ด
- ชื่อเจ้าหน้าที่ถูกทำให้กดได้ชัดเจน แต่ยังคงลุคสุภาพแบบ back-office
- ปุ่ม export ใช้คำและ icon เดียวกับหน้ารายงานอื่น: `พิมพ์`, `PDF`, `CSV`

## Registration
- แยกซ้าย/ขวาแบบง่าย:
  - ซ้าย: ภาพรวมและคำอธิบายระบบ
  - ขวา: ฟอร์มสมัคร
- ในฟอร์มด้านขวาแยก section ชัดเจน:
  - `ข้อมูลทั่วไป`
  - `ข้อมูลเข้าสู่ระบบ`
- ลดความสับสนระหว่างชื่อที่ใช้แสดงในระบบกับ username/password ที่ใช้ล็อกอินจริง

## Admin User Edit Layout
- ซ้ายเป็น `ข้อมูลปัจจุบัน`
  - รูปประจำตัว
  - ลายเซ็น
  - ข้อมูลสรุปสำคัญ
- ขวาเป็น `แบบฟอร์มแก้ไข`
  - ข้อมูลทั่วไป
  - บัญชีผู้ใช้
  - สิทธิ์
  - สื่อประกอบ
  - เปลี่ยนรหัสผ่านแบบแยกส่วน
- แนวคิดคือให้มองเห็นข้อมูลเดิมตลอดเวลา ลดโอกาสแก้ผิดคนหรือแก้ผิด field

## Thai Wording
- ใช้ภาษาไทยเป็นหลักใน title, subtitle, form label, alert message
- คำที่เน้นให้จดจำง่ายและใช้ซ้ำ:
  - `เวรวันนี้`
  - `ลงเวลาเวร`
  - `ตรวจสอบเวร`
  - `จัดการผู้ใช้งาน`
  - `แก้ไขข้อมูลผู้ใช้งาน`

## Name Display Strategy
- แสดงชื่อจาก `first_name + last_name` เมื่อมีข้อมูล
- ถ้ายังเป็นแถวข้อมูลเก่า ให้ fallback ไป `fullname`
- ใช้ helper เดียวใน [auth.php](/C:/xampp/htdocs/staff-main/includes/auth.php) เพื่อลด logic ซ้ำ

## Fallback Avatar / Signature
- ถ้าไม่มีรูปประจำตัว ใช้ placeholder silhouette บนพื้นหลังโทนเดียวกับโรงพยาบาล
- ถ้าไม่มีลายเซ็น แสดงกล่อง placeholder พร้อมข้อความ `ยังไม่มีลายเซ็น`
- การไม่มีรูปหรือลายเซ็นต้องดูเป็นสถานะปกติ ไม่ใช่ error

## Known Limitations
- [profile.php](/C:/xampp/htdocs/staff-main/pages/profile.php) ยังเป็นหน้าที่มีโค้ด legacy เดิมบางส่วนอยู่ แม้แสดงชื่อจาก helper แล้ว
- หากยังไม่รัน migration `007_add_first_last_name_to_users.sql` หน้าสมัครจะยังทำงานได้ แต่การแก้ไขชื่อแบบแยก field ในฝั่งฐานข้อมูลจะยังไม่ครบเต็มรูปแบบ
